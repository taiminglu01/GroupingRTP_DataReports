📖 全方位專案工作流操作手冊

🚀 1. 自動化排程機制 (Automation)

系統具備完整的 End-to-End 自動化能力，透過 Python 與 Batch 腳本串接，實現「一鍵更新、渲染並發布」的無人值守工作流。

  • 一鍵啟動：透過執行 auto_run.bat（內容為 py auto_pipeline.py）啟動整個工作流。
  • 自動替換時間：腳本啟動時，會自動獲取當天日期，並使用正則表達式（Regex）自動將各個 SQL 檔案中的 @endTime 變數修改為 [今日日期] 00:00:00。
  • 執行管線 (Pipeline)：
    1. 呼叫 grafana_query.py 依序執行 4 支 SQL，下載最新 JSON 數據。
    2. 呼叫 compute_retention.py 從原始留存紀錄計算出各玩家等級的 D1/D3/D7 留存率。
    3. 呼叫 render_super_integrated_report.py 融合所有數據，生成視覺化的 HTML 綜合報表。
  • 自動發布 (Git Sync)：HTML 產出後，會自動將 index.html 複製到 GroupingRTP_DataReports 本機數據庫庫，並自動執行 Git 同步與推送。

🔄 2. 數據更新與報表產出流程 (Data Updates)

若不使用自動化腳本，手動更新與除錯的流程如下：

  • 成效數據 (Performance)：執行 py code\grafana_query.py code\summary_analysis_v3.sql 24，重新命名為 old_perf.json 與 new_perf.json。
  • 購買免遊 (Buy Feature)：執行對應 SQL，命名為 buy_data_latest.json。
  • 留存原始數據：執行留存 SQL，命名為 retention_raw.json。
  • 資料二次運算與渲染：執行 py scratch\compute_retention.py，接著執行 py scratch ender_super_integrated_report.py 產出 HTML。

💻 3. 核心數據語法與演算法 (Providing Syntax)

  • CLI 查詢語法：py code\grafana_query.py code\summary_analysis_v3.sql 24 (24代表 GameTower3_Log_TMD)
  • 幣值歸一化：休閒館(04)幣值為 1:10000，計算時必須 / 100。其餘 01/06/07 則為 1:100，直接使用。
  • 總押注 (Total Bet) 計算：嚴格過濾 alterId，僅限 0,1,2,3,4,201，排除 10~15 免費遊戲。
  • RTP 計算防呆：ROUND((1.0 * SUM(win)) / NULLIF(SUM(bet), 0) * 100, 2)。

🛑 4. 專案鐵律與規範 (Constraints)

  • 絕不造假：所有的數字必須 100% 來自資料庫。缺漏強制使用 [待提供]。
  • 數字精確化：嚴禁使用「~」、「約」，必須使用精確數字。
  • 中立性與提示：必須平衡提出優點與缺點風險，並標示為 Agent 建議。
  • 防呆驗證機制：產出報告前必須掃描禁用字眼（如雙軌制、重新映射）。
